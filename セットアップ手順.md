# セットアップ手順

## 1. スプレッドシートを準備する
1. Googleスプレッドシートを新規作成し、ファイル名を `konkatsuapp_selfintroduction_sheet` にする。
2. シート名を `sheet` にする。
3. 1行目に以下のヘッダーをそのまま入力する（列の順番は自由。ヘッダー文字列で列を判定するため）。
   ```
   id  cipherText  encryptedKey  ownerHash  viewerHash  status  schemaVersion  createdAt  updatedAt  firstViewedAt  lastViewedAt  viewCount
   ```
4. 2行目以降は空のままにしておく（GASが書き込みます）。元のExcelにあった「説明文」の行は削除してください。

## 2. Apps Scriptプロジェクトを作成する
1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く。
2. `Code.gs` の中身を、このリポジトリの `Code.gs` の内容に置き換える。
3. 新しいスクリプトファイル `Aes.gs` を追加し、`Aes.gs` の内容を貼り付ける。

## 3. LINEログインチャネルの設定
LIFFアプリで `liff.getIDToken()` を使えるように、LINE Developersコンソールで該当のLIFFアプリ（チャネル）のスコープに **openid** が含まれていることを確認してください（含まれていない場合はIDトークンが取得できません）。

## 4. スクリプトプロパティを設定する
1. Apps Scriptエディタの左メニュー「プロジェクトの設定」→「スクリプト プロパティ」を開く。
2. 一度、エディタ上部の関数選択で `generateSecrets_` を選んで実行し（初回は権限の承認が必要です）、実行ログ（表示 → ログ）に出力された `AES_KEY_B64` と `HASH_SALT` の値をコピーする。
3. 以下の4つのプロパティを登録する。

   | プロパティ名 | 値 |
   |---|---|
   | `SPREADSHEET_ID` | スプレッドシートのURLの `/d/` と `/edit` の間の文字列 |
   | `AES_KEY_B64` | 手順2でログに出力された値 |
   | `HASH_SALT` | 手順2でログに出力された値 |
   | `LIFF_CHANNEL_ID` | LIFF IDの「-」より前の数字部分（例：`2010606389-v29ZSV0f` なら `2010606389`） |

4. 動作確認として `aesSelfTest_`（Aes.gs内）を実行し、ログに `AES self-test OK` と出ることを確認する。

## 5. ウェブアプリとしてデプロイする
1. 「デプロイ」→「新しいデプロイ」→種類の選択で「ウェブアプリ」を選ぶ。
2. 「次のユーザーとして実行」：自分
3. 「アクセスできるユーザー」：全員
4. デプロイ後に発行される「ウェブアプリのURL」（`https://script.google.com/macros/s/xxxx/exec`）をコピーする。

## 6. アプリ側の設定
`app.js` 冒頭の以下の値を、手順5で取得したURLに置き換えてください。
```js
const WEB_APP_URL = "https://script.google.com/macros/s/XXXXXXXXXXXXXXXXXXXXXXXXXXXX/exec";
```

## 7. 動作確認
- 本人の端末で「共有する」→ LINEの共有画面が出て送信できること
- 送られたリンクを別のLINEアカウントで開いたとき、内容が表示されること（1人目）
- 同じリンクをさらに別の3人目のアカウントで開いたとき、内容が表示されずブロックされること
- 「設定」タブの「共有リンクを無効化する」を押した後、そのリンクを開くと無効化メッセージが出ること

## 補足：この設計でできること・できないこと
- **できること**：スプレッドシートを直接開いても、LINEユーザーIDは元に戻せない形（HMAC-SHA256、秘密saltはスクリプトプロパティのみに保存）で、回答内容もAES-256で暗号化された状態でしか見えません。共有リンクは最初に開いた本人以外弾かれるため、転送されても中身は見えません。
- **前提として注意していただきたい点**：Apps Scriptプロジェクトの編集権限（スクリプトプロパティを見られる権限）を持つ人は、鍵とsaltを取得できてしまうため、理論上は復号やLINEユーザーID特定が可能です。「管理者=スプレッドシートを見られる人」と「開発者=Apps Scriptを編集できる人」を分けて運用し、後者を必要最小限にすることをおすすめします。
